# photo-memory-spike — Claude 向けプロジェクト指示

## このプロジェクトの性質

- **spike**。捨ててよい。きれいさより「あの感じ」に最速で触ることが優先。
- ローカル完結 (写真は端末から出さない、IndexedDB 保存)、バックエンドなし。
- スコープ境界は README.md と TODO.md の冒頭を参照。**迷ったら「これは A (今すぐ) か?」で判定**。

## ファイル構成

```
index.html              # 本体 (HTML + CSS + JS インライン、CDN 依存)
TODO.md                 # 「次に何をやる」の生きたリスト + 現在地
CHANGELOG.md            # 「過去に何をやったか」の蓄積 (背景・設計判断・教訓)
README.md               # 外向き説明、ほぼ更新しない
.claude/launch.json     # ローカル dev サーバ設定 (port 5273、別プロジェクトと衝突回避)
.gitignore
```

ホスティングは GitHub Pages (`https://yutsutke.github.io/photo-memory-spike/`)。リポは public。

## 🔁 セッション終了時の必須ルール

**実装作業をした session を閉じる前に、必ず TODO.md と CHANGELOG.md を更新する。**

これは次セッション (別の Claude / 別の日の自分) が状況復旧するための引き継ぎコスト最小化。spike だからこそ口頭の文脈が失われやすく、文書が唯一の memory になる。

### TODO.md の更新内容

- **「現在地」セクションを書き換える** — 何が ✅ クローズで、次の ▶ 着手前は何か、を一目で示す
- 完了したチェックボックスに `[x]` を入れる
- 触っているうちに**新しく見えてきた道**は TODO の末尾 or 「Phase X」セクションに追記
- ユーザーが「これはやらない」と言ったものは「今は作らない」リストに移す

### CHANGELOG.md の更新内容

新しいバージョン (`vN`) を**先頭に追加** (最新が上)、以下の構造で記述:

```markdown
## vN — 一行サマリ (YYYY-MM-DD)

**背景**
- なぜこの作業に着手したか、何を確かめたかったか

**設計判断**
- 採用した方針と、その理由
- 採用しなかった代替案 (あれば)

**ハマったところ** (あれば)
- 症状 → 原因 → 対処の順で具体的に
- 次の Claude が同じ罠を踏まないように

**結果 / 観察**
- 何が動いたか、ユーザーのフィードバック (生の言葉が強い signal)
- 触ってみて分かった signal

**教訓**
- このプロジェクト固有の学び (一般論ではなく)

**残課題 / 次の方向**
- 持ち越しタスク、次セッションで考えること
```

### 何を書かないか

- diff や行番号レベルの細かい変更内容 (git log で見られる)
- ありきたりな一般論 (spike 固有の学びだけ)

## 🏷 BUILD バージョン文字列の運用

`index.html` 上部に `const BUILD = 'phaseN.M ...';` がある。**ヘッダ右に画面表示**される (キャッシュ問題切り分け用)。

- **commit ごとに必ず上げる** (例: `phase1.10` → `phase1.11`)
- iOS Safari の HTML キャッシュは強烈。これが表示されないと「新しい版が来てない」とすぐ気付ける
- spike 中はこの運用を絶対に維持する

---

## 🚨 iOS Safari のハマりどころ (過去にハマった)

- **HTML キャッシュ強烈**: HTML に no-cache メタタグ必須。URL `?v=N` で強制リフレッシュも併用
- **`createImageBitmap` が TypeError を投げるバージョンがある** — 古典 `<img>` + `<canvas>` + `URL.createObjectURL` の方が堅い
- **同じ名前の関数を二つ書くと事故** (`makeThumb` 衝突) — 別責務には別名
- **canvas/Blob 系のメモリ管理が緩い** — 大量の `getImageData` で表示中の Blob URL が無効化される。縮小描画でメモリ最小化、完了時に URL キャッシュ revoke + 再描画
- **フォトピッカーは HEIC を裏で JPEG 変換**。HEIC のまま渡したいなら「ファイル」アプリ経由
- **`e.target.value = ''` を await の前に呼ばない** — File 参照が無効化される疑い、`finally` で

詳細は CHANGELOG v3-v5 を参照。

## スコープを膨らませない歯止め

ユーザーが触っている中で出てきた要望は強い signal だが、TODO の「今は作らない」リストに該当するものは安易に着手しない。

特に:
- 地図ビュー
- プラットフォーム化 (汎用カーネル)
- 積極的 curation (☆ お気に入り)
- 認証・クラウド同期・シェア
- proactive 通知 (push)

これらは reminiscence の手触り検証が終わってから次の周回で。

## 「情報量を増やす提案」は毎回吟味する

UI に情報を増やす変更を提案する時は、`memory/ui-minimalism-works.md` (Claude の memory) の指針に照らす:

> 「これは思い出しの意識を写真から逸らさないか?」

デフォルト体験は最小限のまま、追加は「使う時だけ広がる」形に倒す。
