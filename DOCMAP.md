# 資料マップ (DOCMAP) — どこに何があるか / いつ見るか

> このプロジェクトの資料は **2 系統**。**Notion = これから作るプロダクトの方針** / **この git = spike の検証アーカイブ (作って確かめたもの)**。
> セッション開始時にこれを読んで地図を掴む。迷ったら下の「迷ったらどこを見る?」へ。

---

## 🧭 プロダクトのコア (全判断の起点)

- **コア (判断軸)**: 偶然よみがえった久しぶりの記憶
- **体験コピー (外向き・起動画面/ストア)**: 偶然あの時に久しぶりに戻る
- **3 要素**: ① 偶然 ② 久しぶり ③ よみがえる
- **名前**: 🇬🇧 Madeleine / 🇯🇵 あの日（タグライン「写真と足跡から蘇る」、2026-06-17 変更。旧: 写真思い出）

> 機能追加・広告・AI・名前など迷ったら、**この 3 要素を強めるか / 壊すか**で判断する。「何かが始まる」(次の旅・行動) は派生効果であってコアではない。

## 迷ったらどこを見る?

| 知りたいこと | 見る場所 |
|---|---|
| プロダクトのコア・軸 | この冒頭 / Notion WHY §0・§2 |
| 次に何を作る (実行順) | Notion HOW §8 Phase 0-6 |
| de-risk の観点 (何を先に潰す) | Notion WHY §4 |
| いくらかかる / どう出す | Notion HOW (コスト・署名・ストア) |
| spike で何を確かめた / なぜこの実装か | git CHANGELOG / TODO |
| コードのどの機能を残す/変える/捨てるか | Notion WHY §1.5 (資産移行マップ) |

## 更新ルール

- **プロダクト方針が変わった** → Notion (WHY/HOW) を更新
- **spike の実装をいじった** → git TODO.md / CHANGELOG.md (CLAUDE.md の必須ルール)
- **資料の構成自体が変わった** → この DOCMAP.md を更新
